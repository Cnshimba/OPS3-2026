The Container Paradigm

"Virtual Machines isolate the Hardware. Containers isolate the Process Space."

  • Shared Kernel: All containers run on the host's kernel.
  • Isolation Principles:
    • Namespaces: Isolate processes, network, mounts (What you see).
    • Cgroups: Limit CPU, Memory, I/O (What you use).
  • Efficiency:
    • VM: GBs of RAM, Minutes to boot.
    • Container: MBs of RAM, Seconds to boot.

System Containers: LXC CLI

Before a GUI, there was the Command Line.

Creating a Container:

# -n (Name), -t (Template)
sudo lxc-create -n my-web -t download -- --dist ubuntu ...

Lifecycle Management:

sudo lxc-ls --fancy       # List State & IPs
sudo lxc-start -n my-web  # Boot Init System
sudo lxc-stop -n my-web   # Graceful Shutdown

Access (lxc-attach):

# Enter the container namespace as root (No SSH needed)
sudo lxc-attach -n my-web
  root@my-web:~# apt install apache2

LXC in Proxmox VE (GUI)

Proxmox abstracts LXC complexity into a VM-like interface.

  • Templates:
    • Download OS images (Ubuntu, Alpine) via Storage View.
  • Unprivileged Containers:
    • Default & Recommended. Maps Container Root (UID 0) to unprivileged host user. Enhances security.
  • Dynamic Resources:
    • Elasticity: Increase RAM or CPU limits while the container is running.
    • Zero downtime scaling.

Application Containers: Docker

Docker packages applications + dependencies into portable images.

Architecture:

  • Daemon (dockerd): Background service (Runs as Root).
  • CLI: Sends API commands to daemon.
  • Image Layers: Read-only templates; reused for speed.

Workflow:

# Pull -> Create -> Start (Detached) -> Map Ports
docker run -d --name web -p 8080:80 nginx

docker ps           # List Running
docker logs web     # View Application Output
docker exec -it web bash  # Debugging Shell

Podman: The Secure Alternative

Daemonless & Rootless

  • No background daemon required.
  • Runs as User Process (Added Security).
  • Pod Support: Manage groups of containers sharing localhost (Kubernetes styled).
# Syntax is compatible with Docker
podman run -d --name rootless-web -p 8081:80 nginx

# Create a Pod
podman pod create --name my-stack -p 8888:80

Summary & Comparison

Feature System (LXC) Application (Docker/Podman)
Role "Lightweight VM" "Single Service"
Processes Multiple (Init, Cron, SSH) One (Entrypoint)
State Persistent Ephemeral
Management lxc / Proxmox GUI docker / podman CLI
Use Case Infrastructure (VPN, DNS) Microservices
