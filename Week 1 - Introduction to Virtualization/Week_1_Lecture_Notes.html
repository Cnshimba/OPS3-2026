3. What is Virtualization?

Virtualization is the process of creating a software-based (or "virtual") representation of something, rather than a physical one. It allows you to run multiple operating systems simultaneously on a single physical machine.

Core Concept: Abstraction

Virtualization abstracts the physical hardware (CPU, RAM, Storage, Network) from the operating system. A software layer called a Hypervisor handles this abstraction.

Why Virtualize? (Benefits)

  1. Server Consolidation: Run multiple services (Web, DB, Mail) on one physical server instead of three, reducing hardware costs and energy consumption.
  2. Isolation: If one VM crashes or gets hacked, it doesn't affect the others.
  3. Flexibility & Agility: Create, delete, or move VMs in minutes.
  4. Legacy Support: Run older OS versions on modern hardware.
  5. Disaster Recovery: VMs are just files (images); they are easy to back up and restore.

4. Hypervisor Architecture

The Hypervisor (or Virtual Machine Monitor - VMM) is the software that sits between the hardware and the virtual machines. It creates and runs VMs.

Type-1: Bare-Metal Hypervisors

  • Runs directly on: Physical Hardware.
  • No underlying OS: The hypervisor is the OS.
  • Performance: High performance, access to raw hardware resources.
  • Use Case: Enterprise datacenters, production servers.
  • Examples: Proxmox VE, VMware ESXi, Microsoft Hyper-V, Xen.

Type-2: Hosted Hypervisors

  • Runs on: A standard Operating System (Host OS) like Windows, Linux, or macOS.
  • Overhead: Higher overhead because it goes through the Host OS to reach hardware.
  • Use Case: Personal desktops, testing, development (e.g., trying Linux on Windows).
  • Examples: Oracle VirtualBox, VMware Workstation.
Feature Type-1 (Bare-Metal) Type-2 (Hosted)
Installation Installed directly on hardware Installed as an app on an OS
Performance High (Near-native) Moderate (OS overhead)
Efficiency High resource efficiency Lower (Host OS takes resources)
Example Proxmox VE, ESXi VirtualBox

5. Virtual Machine Anatomy (Concepts)

Before we build VMs, we must understand their components:

5.1 vCPU (Virtual CPU)

  • Concept: The hypervisor schedules vCPUs on physical cores (time-slicing).
  • Rule: Never allocate more vCPUs than you have physical threads.

5.2 vRAM (Virtual RAM)

  • Concept: A dedicated block of physical RAM assigned to the VM.
  • Rule: RAM cannot be time-sliced efficiently. Do not over-commit (e.g., don't assign 20GB RAM on a 16GB server).

5.3 vDisk (Virtual Disk)

  • Concept: The "hard drive" is just a file on the host (e.g., vm-100-disk-0.qcow2).
  • Formats:
    • RAW: Fast, pre-allocated.
    • QCOW2: Thin-provisioned (grows as used), supports snapshots.

5.4 vNIC (Virtual Network Interface)

  • Concept: A software network card connecting to a virtual switch (Linux Bridge).
  • Types:
    • E1000: Compatible (emulated).
    • VirtIO: High performance (paravirtualized).

6. KVM and QEMU

Proxmox VE relies heavily on these two technologies.

KVM (Kernel-based Virtual Machine)

  • What is it? A kernel module that turns the Linux kernel into a Type-1 hypervisor.
  • Role: It allows the kernel to act as a hypervisor, handling CPU and memory virtualization directly.
  • Comparison: Because Linux runs directly on hardware, KVM allows VMs to run at near-native speed.

QEMU (Quick Emulator)

  • What is it? An emulator that provides hardware virtualization.
  • Role: While KVM handles the CPU/RAM, QEMU emulates the hardware devices the VM "sees" (Hard drive, Network card, USB controller, Graphics card).
  • Together: KVM + QEMU = A powerful, open-source Type-1 virtualization solution.

7. Introduction to Proxmox VE

Proxmox Virtual Environment (VE) is an open-source server virtualization management platform.

Key Features:

  • Based on: Debian Linux.
  • Hypervisor: Uses KVM for Virtual Machines and LXC for Containers.
  • Management: 100% Web-based interface (GUI). No need to install a separate management client.
  • Enterprise Features: High Availability (HA), Clustering, Live Migration, Backups.
  • Cost: Free to use (AGPL v3 license), with paid enterprise support subscription.

Comparison: Proxmox vs. The Rest

  • vs. VMware ESXi: Proxmox is open-source and free; ESXi has valid free limits and is proprietary. Proxmox supports LXC containers natively.
  • vs. VirtualBox: Proxmox is a Type-1 OS for servers; VirtualBox is a Type-2 app for your laptop.
