Automation and Cloud API

Week 11

Operating Systems 3 (Virtualisation & Cloud Technologies) Instructor: KT Nshimba

Learning Objectives

  • Automation: Why we stop clicking buttons ("Pets vs Cattle").
  • Tools: Cloud-Init, Heat, Ansible, Magnum.
  • Methodology: Imperative (Scripting) vs. Declarative (IaC).
  • Labs: 4-Part "Zero to Hero" automation series.

1. Why Automate?

"Cattle, not Pets."

  • Pets: Distinct, named (e.g., "Gandalf"), manually nursed to health.
  • Cattle: Identical, numbered (e.g., "vm-001"), replaced when sick.

To manage scale (Cattle), you need Robots (Automation).

2. Cloud-Init (The Bootstrap)

How do we configure a VM before we can SSH into it?

  1. Launch: Pass a YAML file (#cloud-config) to the API.
  2. Metadata Service: OpenStack hosts the file at 169.254.169.254.
  3. Boot Stages: The VM wakes up, finds the Datasource, and executes instructions (Init -> Config -> Final).

3. Approaches to Infrastructure

Feature Imperative (Bash/CLI) Declarative (Heat/Terraform)
Focus "How" (Steps) "What" (End State)
Example run server, sleep 5, assign ip "I want 1 Server with 1 IP"
Idempotency Difficult (Script breaks on re-run) Native (Tool calculates delta)
Use Case Quick tasks, Glue code Production Infrastructure

4. Heat (Infrastructure as Code)

Heat Orchestration Templates (HOT) define the stack.

  • Resources: Network, Security Group, Server, Volume.
  • Dependencies: Heat builds them in the correct order (Net -> Port -> Server -> IP).
  • Updates: Changing the template updates the live infrastructure (Self-Healing).

5. Ansible (Configuration Management)

Heat builds the House (Infrastructure). Ansible installs the Furniture (Software).

  • Inventory: List of servers (hosts.ini).
  • Playbook: YAML file describing tasks.
  • Agentless: Uses SSH, no software needed on the target.
- name: Install Apache
  hosts: webservers
  tasks:
    - apt: name=apache2 state=present

6. Magnum (Container Platform)

Moving up the abstraction ladder: Pods > VMs.

  • Magnum: OpenStack service that deploys Container Engines.
  • Workflow:
    1. Create Template (Fedora CoreOS).
    2. Provision Cluster (Heat builds Masters/Workers).
    3. Configure kubectl.
    4. Deploy App (Nginx Pods + Load Balancer).
