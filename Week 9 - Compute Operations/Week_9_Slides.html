Week 9: Compute Operations (Nova)

Operating Systems 3 (Virtualisation & Cloud Technologies) Instructor: KT Nshimba

1. Introduction to Nova

  • Role: The "Compute Controller" of OpenStack.
  • Function: Orchestrates hypervisors (KVM/QEMU) to spawn instances.
  • Industry Context:
    • OpenStack: Nova acts as the "Microsoft.Compute" Resource Provider.
    • Azure: You talk to ARM -> ARM talks to Microsoft.Compute.
    • AWS: Equivalent to the EC2 Service.
  • Key Concept: Nova is the Factory, the Instance is the Product.

2. Nova Architecture (The Anatomy)

  • API (nova-api): The Front Door. Accepts REST requests. Stateless/Scalable.
  • Scheduler (nova-scheduler): The Brain. Decides where a VM lands.
  • Conductor (nova-conductor): The Guard. Protections the database from Compute nodes.
  • Compute (nova-compute): The Worker. Runs on the Hypervisor. Talks to Libvirt/KVM.
  • Placement: Tracks inventory (RAM/CPU available).

3. The Scheduling Algorithm

How does Nova choose 1 host out of 1,000?

  1. Pass 1: Filtering (Qualifying)

    • RamFilter: Does it have enough RAM?
    • ComputeFilter: Is the host alive?
    • ImagePropertiesFilter: Does it match the OS requirements?

    • Pass 2: Weighting (Ranking)

    • RamWeigher:

      • Spread: Pick the emptiest host (Performance).
      • Stack: Pick the fullest host (Power Saving).

4. Instance Lifecycle

The State Machine of a VM:

  1. BUILD: API accepted, Scheduler finding a home.
  2. Networking: Conductor asks Neutron for an IP.
  3. Spawning: Hypervisor downloads disk from Glance.
  4. ACTIVE: VM is running successfully.
  5. ERROR: Something went wrong (Check nova-compute.log).
  6. SHELVED: Offloaded to disk (Deep Freeze).

5. Operations: Flavors & Keys

  • Flavors (Capacity)
    • Virtual Hardware Templates (vCPU, RAM, Disk).
    • AWS/Azure: You Select (Consumer).
    • OpenStack: You Create (Provider).
    • Naming: m1.nebula_micro (Class.Family.Size).
  • Keypairs (Access)
    • Concept: Lock (Public) and Key (Private).
    • Mechanism: Nova injects 'Lock' into VM. Laptop uses 'Key' to decrypt challenge.
    • Result: Secure, password-less login.

6. Operations: Security Groups

  • The Virtual Firewall
    • Applied at the vNIC level (Hypervisor).
    • Stateful: Return traffic automatically allowed.
    • Allow-List: Implicit Deny (Block All) by default.
  • Implementation
    • Neutron talks to OVS Agent.
    • OVS Agent writes to Flow Tables on the wire.
    • Traffic dropped before reaching the VM.
