Week 6 - Lab 4: Shared Storage (NFS)

Module: Operating Systems 3 (Virtualisation & Cloud Technologies)
Topic: Network Attached Storage (NFS) and Proxmox Backends
Estimated Time: 60 Minutes
Instructor: KT Nshimba

Lab Overview

In a real datacenter, Proxmox nodes rarely store data on their own hard drives. They use massive external storage arrays (SAN/NAS) so that if a node dies, the data is safe elsewhere. In this lab, you will simulate this environment. You will convert a standard Linux VM into a "Storage Appliance" (NFS Server) and connect your Proxmox host to it.

Objectives:

  1. Configure a Linux VM to act as an NFS Server.
  2. Mount the remote NFS share into Proxmox VE as a Storage Backend.
  3. Migrate or store data on the shared storage.

Prerequisites: - Your Ubuntu Server VM (VM 200). - Proxmox VE Host.

Part 1: Building the NAS (Network Attached Storage)

We will turn your Ubuntu VM into a file server.

  1. Log in to VM 200 (SSH or Console).
  2. Install NFS Server: bash sudo apt update sudo apt install nfs-kernel-server -y
  3. Create the Shared Folder: bash sudo mkdir -p /srv/nfs/iso # Set permissions so anyone can read/write (For Lab Only! Insecure for Prod) sudo chown nobody:nogroup /srv/nfs/iso sudo chmod 777 /srv/nfs/iso
  4. Export the Share:
    • Edit the exports file: sudo nano /etc/exports
    • Add this line at the bottom: bash /srv/nfs/iso *(rw,sync,no_subtree_check)
    • Explanation: * means allow any IP. rw means Read-Write.
  5. Apply Configuration: bash sudo exportfs -a sudo systemctl restart nfs-kernel-server
  6. Field Report:
    • NFS Server IP Address: [ _____________________________ ]
    • Export Path: /srv/nfs/iso

Part 2: Connecting Proxmox to the NAS

Now we tell Proxmox to use this "External Storage".

  1. Open Proxmox GUI.
  2. Navigate to Datacenter -> Storage.
  3. Click Add -> NFS.
  4. Form Details:
    • ID: lab-nfs (The name Proxmox will use).
    • Server: <IP_OF_YOUR_UBUNTU_VM> (e.g., 192.168.

      • Click on lab-nfs in the left tree.
      • Select ISO Images.
      • Upload a small file (e.g., the Alpine Linux ISO from Week 1).

    • Verify on the Server:

      • Go back to the Ubuntu VM terminal.
      • Check the folder: bash ls -lh /srv/nfs/iso/template/iso/
      • Observation: You should see the file you just uploaded from Proxmox!

    • Backups:

      • Go to Datacenter -> Backup.
      • Create a backup job.
      • Select Storage: lab-nfs.
      • Run it.
      • Result: Your VM backup is now physically located inside the other VM. (In reality, this would be a separate physical server).

Part 4: Critical Simulation (The Crash)

What happens if the storage server dies?

  1. Shutdown the NAS:
    • In Proxmox, power off VM 200 (The Ubuntu VM).
  2. Observe Proxmox:
    • Click on lab-nfs storage.
    • It will likely hang or show a question mark.
    • Try to list ISOs. It will fail (timeout).
  3. Lesson: Proxmox is now waiting for storage that doesn't exist. This is why Storage Availability is critical.
  4. Restore:
    • Start VM 200.
    • Refresh Proxmox. lab-nfs should come back online.

Lab Checkpoint

  • I installed nfs-kernel-server on my Linux VM.
  • I successfully mounted the NFS share in Proxmox.
  • I verified that files uploaded via Proxmox config appear on the Linux filesystem.

Reflection: * Why is NFS preferred over "Local Directory" for Clusters? * Why did Proxmox "hang" when we turned off the VM?
